Commands used:

git checkout -b feature/user-authentication
git add auth.js
git commit -m "Add basic auth logic"
git add package.json
git commit -m "Add jwt dependency"
git commit -m "Fix typo in auth file"
git add middleware.js
git commit -m "Add middleware for authentication"
git add tests.js
git commit -m "Add basic auth tests"
git rebase -i HEAD~5
git checkout main
git commit -m "Update main branch"
git checkout feature/user-authentication
git rebase main

# Before rebase (git log --oneline)

e5f6g7 Add basic auth tests
d4e5f6 Add middleware for authentication
c3d4e5 Fix typo in auth file
b2c3d4 Add jwt dependency
a1b2c3 Add basic auth logic

# After rebase (git log --oneline)

e5f6g7 Add basic auth tests
e6f7g8 Add middleware for authentication
b2c3d4 Add jwt dependency and fix typo
a1b2c3 Add basic auth logic

# Explanation:

Rebase is used to clean up and rewrite commit history before merging into main.
Merge keeps all commits (including messy ones) and creates a merge commit.
Rebase makes the history linear and easier to read, ideal before pushing feature branches.
